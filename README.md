# Constraint-Aware Hierarchical Search

An LLM-powered hierarchical regulation tree search and QA system.

---

## Quick Start

### 1. Install

```bash
conda create -n constraint-search python=3.11 -y
conda activate constraint-search
pip install -r requirements.txt
```

### 2. Configure

```bash
cp .env.example .env
```

Edit `.env` with your API credentials:

```env
OPENAI_CHAT_BASE_URL=https://api.openai.com/v1
OPENAI_CHAT_API_KEY=sk-...
OPENAI_MODEL=gpt-4o
OPENAI_EMBEDDING_BASE_URL=https://api.openai.com/v1
OPENAI_EMBEDDING_API_KEY=sk-...
OPENAI_EMBEDDING_MODEL=text-embedding-3-small
```

### 3. Prepare Data

Place your regulation text files in `input/` (`.txt` format).

### 4. Generate Rules

Different regulation documents vary in hierarchical structure, coding format, and chunking granularity. This step asks the LLM to analyze your text samples, automatically identify numbering rules, hierarchy depth, exclusion patterns, etc., and generate extraction rules accordingly — resulting in better tree quality.

```bash
python scripts/analyze_and_write_rules.py --dataset-name my_dataset --input-path input/
```

Output goes to `rules/my_dataset/`.

### 5. Build Tree Index

```bash
python scripts/regtree_cli.py build --dataset-name my_dataset
```

Output goes to `rag-storage/my_dataset/`. Add `--resume` to continue an interrupted build from checkpoint.

### 6. Query

```bash
python scripts/regtree_cli.py query --dataset-name my_dataset "titanium dioxide; TiO2 content >= 80%"
```

## Algorithm

### Tree Construction (H)

Prompt definition: [`tree_index.py:51-73`](regtree_agent/tree_index.py#L51) · [with rules injection → `tree_index.py:876-911`](regtree_agent/tree_index.py#L876)

> **rules injection**: `hierarchy_hints` (hierarchy encoding rules + few-shot extraction examples) and `code_format_hint` (code field format specification) are both from [`rules/<dataset>/rule_profiles.json`](rules), automatically generated by LLM analyzing text samples. At runtime they are appended to `CHUNK_EXTRACT_TASK` as rule #8.
>
> Different datasets have different coding systems, hierarchy depths, and numbering formats, so the content of `hierarchy_hints` and `code_format_hint` varies by dataset. The benefit: no manual prompt engineering needed — the LLM analyzes a few samples to automatically adapt to new regulatory documents; few-shot examples are derived directly from the source text, ensuring code concatenation and children nesting are consistent with the actual dataset, improving tree construction quality.
>
> Taking the import-export tariff (进出口税则) as an example, `hierarchy_hints` contains:
> ```
> **层级规则：**
> - 第1层（章）：2位数字，范围01-97
> - 第2层（品目）：4位数字，格式为"章.品目号"（如06.01、20.01）
> - 第3层（子目）：6位或8位数字，格式为"品目号.子目号"（如06.01.10、20.01.90）
> - 第4层（子子目）：部分子目下用字母+数字细分（如a.1、c.2等）
>
> **示例1 - 2层嵌套（章→品目）：**
> 原文：第六章 活树及其他活植物；鳞茎、根及类似品；插花及装饰
>       06.01   鳞茎、块茎、块根、球茎、根颈及根茎，休眠、生长或开花的；菊苣植物及其根，但品目 12.12 的根除外：
> 提取结果：{code:"06", title:"第六章 活树及其他活植物...", children:[{code:"06.01", title:"鳞茎、块茎..."}]}
>
> **示例2 - 3层嵌套（章→品目→子目）：**
> 原文：06.01   鳞茎、块茎、块根、球茎、根颈及根茎，休眠、生长或开花的；菊苣植物及其根，但品目 12.12 的根除外：
>          10 —  休眠的鳞茎、块茎、块根、球茎、根颈及根茎
>          20 —  生长或开花的鳞茎、块茎、块根、球茎、根颈及根茎；菊苣植物及其根
> 提取结果：{code:"06.01", title:"鳞茎、块茎...", children:[{code:"06.01.10", title:"休眠的鳞茎、块茎..."}, {code:"06.01.20", title:"生长或开花的鳞茎..."}]}
>
> **示例3 - 4层嵌套（章→品目→子目→子子目）：**
> 原文：06.03   制花束或装饰用的插花及花蕾，鲜、干、染色、漂白、浸渍或用其他方法处理的：
>            —   鲜的：
>         11 — — 玫瑰
>         12 — — 康乃馨
>         13 — — 兰花
>         19   — — 其他
>         90   —   其他
> 提取结果：{code:"06.03", title:"制花束或装饰用的插花...", children:[{code:"06.03.11", title:"玫瑰"}, {code:"06.03.12", title:"康乃馨"}, {code:"06.03.13", title:"兰花"}, {code:"06.03.19", title:"其他"}, {code:"06.03.90", title:"其他"}]}
>
> **示例4 - 含字母分支的嵌套：**
> 原文：29.08   酚及酚醇的卤化、磺化、硝化或亚硝化衍生物：
>            —   仅含卤素取代基的衍生物及其盐：
>         11 — — 五氯苯酚（ISO）
>         19 — — 其他
>            —   其他：
>         91 — — 地乐酚（ISO）及其盐
>         99 — — 其他
> 提取结果：{code:"29.08", title:"酚及酚醇的卤化...", children:[{code:"29.08.11", title:"五氯苯酚（ISO）"}, {code:"29.08.19", title:"其他"}, {code:"29.08.91", title:"地乐酚（ISO）及其盐"}, {code:"29.08.99", title:"其他"}]}
>
> **示例5 - 注释项处理：**
> 原文：注释：
>   一、除品目 06.01 的菊苣植物及其根以外，本章只包括通常由苗圃或花店供应为种植或装饰用的活树及其他货品...
> 总注释：
>   本章包括由苗圃（包括园林）或花店供应的适于种植或装饰用的各种活植物...
> 提取结果：{code:"06-注释", title:"注释", children:[{code:"06-注释一", title:"一、除品目 06.01 的菊苣植物及其根以外..."}]}
> ```
>
> `code_format_hint` 注入到 `output_schema.units[].code` 字段描述中：
> ```
> code字段采用HS编码体系：
> 1. 章编码：2位数字（01-97），如"06"、"20"、"29"
> 2. 品目编码：4位数字，格式"XX.XX"，如"06.01"、"20.01"、"29.08"
> 3. 子目编码：6位或8位数字，格式"XX.XX.XX"或"XX.XX.XX.XX"，如"06.01.10"、"89.01.10"、"97.01.21"
> 4. 子子目编码：在子目后加字母或数字，如"29.08.a"、"29.08.c.1"
> 5. 注释编码：使用"XX-注释"、"XX-总注释"、"XX-子目注释"格式，如"06-注释"、"06-总注释"
>
> **具体示例：**
> - "06" = 第六章
> - "06.01" = 鳞茎、块茎、块根等
> - "06.01.10" = 休眠的鳞茎等
> - "29.08" = 酚及酚醇的卤化、磺化、硝化或亚硝化衍生物
> - "29.08.11" = 五氯苯酚（ISO）
> - "97.01.21" = 超过100年的油画、粉画及其他手绘画
> ```
