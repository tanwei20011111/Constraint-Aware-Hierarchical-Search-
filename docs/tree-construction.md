# Tree Construction (H)

Prompt definition: [`tree_index.py:51-73`](../regtree_agent/tree_index.py#L51) · [with rules injection -> `tree_index.py:876-911`](../regtree_agent/tree_index.py#L876)

`hierarchy_hints` and `code_format_hint` are loaded from `rules/<dataset>/rule_profiles.json`, which is automatically generated by an LLM from text samples. At runtime, these rules are appended to `CHUNK_EXTRACT_TASK` as rule #8.

Different datasets have different coding systems, hierarchy depths, and numbering formats, so the content of `hierarchy_hints` and `code_format_hint` varies by dataset. This removes manual prompt engineering: the LLM analyzes a few samples to adapt extraction rules to new regulatory documents, while few-shot examples derived from the source text keep code concatenation and child nesting consistent with the dataset.

For the import-export tariff example, `hierarchy_hints` describes:

- Level 1, chapter: 2 digits, range `01-97`
- Level 2, heading: 4 digits, format such as `06.01` or `20.01`
- Level 3, subheading: 6 or 8 digits, format such as `06.01.10` or `20.01.90`
- Level 4, deeper branches: letter and number subdivisions such as `a.1` or `c.2`

`code_format_hint` is injected into the `output_schema.units[].code` field description. It specifies HS-style code formats such as `06`, `06.01`, `06.01.10`, `29.08.11`, and note codes such as `06-注释` or `06-总注释`.
